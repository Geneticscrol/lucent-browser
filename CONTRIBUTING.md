# Contributing to Lucent Browser

Thank you for your interest in contributing to Lucent Browser! This document provides guidelines for contributing.

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.

## How to Contribute

### Reporting Bugs

1. Check if the bug is already reported in [Issues](https://github.com/Geneticscrol/lucent-browser/issues)
2. If not, create a new issue with:
   - Clear title describing the problem
   - Steps to reproduce
   - Expected vs actual behavior
   - System information (OS, version, etc.)
   - Screenshots if applicable

### Suggesting Features

1. Check existing [Issues](https://github.com/Geneticscrol/lucent-browser/issues) and [Discussions](https://github.com/Geneticscrol/lucent-browser/discussions)
2. Create a new issue with:
   - Clear description of the feature
   - Use case / why it's needed
   - How it aligns with Lucent's privacy/transparency goals

### Submitting Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

## Development Setup

See [BUILD.md](docs/BUILD.md) for detailed build instructions.

## Pull Request Guidelines

### Before Submitting

- [ ] Code follows existing style
- [ ] All tests pass
- [ ] Documentation updated if needed
- [ ] Commit messages are clear

### PR Description Should Include

- What changes were made
- Why the changes were needed
- Any breaking changes
- Related issue numbers
